# 快速入门

Previously 用一条垂直时间轴上的单一连续关系取代了聊天线程——你只需出现并开始交谈，代理会针对每条消息自行决定哪些过往上下文是相关的。

## 界面

页面是一个垂直滚动区域，从上到下分为三个部分：

1. **全屏英雄区** —— `"Previously on {你的名字}"` 以电影式文字渐显动画呈现，下方附有 GitHub 署名链接。
2. **记忆时间轴** —— 你过去的切片，最旧在上、最新在下，按日期分组，带有每条切片的时间戳，以及用于加载更早历史的"加载更多"按钮。
3. **实时聊天消息** —— 正在进行的对话，实时流式呈现。

在三者之下，**固定在屏幕底部**，是一个粘性输入条：一个从 24 px 到 72 px 最大高度自动调整的文本区域。按 Enter 提交；按 Shift+Enter 换行。当代理正在流式输出时，提交箭头变为方形停止按钮。工具栏提供图片附件（粘贴、拖放或文件选择器；仅限 `image/*` 类型，每文件 10 MB）、主题切换、语言环境切换和设置链接。

整个布局位于一个 `MessageScroller` 内部——向上滚动回顾过去，向下滚动继续你离开的地方。

> **关键要点：没有会话列表，没有线程管理，没有"新建聊天"按钮。时间轴本身就是会话列表。**

## 记忆如何在你交谈时形成

记忆不是你需要管理的东西——它在你交谈时自动形成。

**切片（slice）** 是一次对话爆发。它在你开始输入时打开，在你参与期间保持活跃，并在静默 30 分钟后关闭。只有这一条规则。没有容量限制，没有话题转移检测，没有 ML 驱动的错误分割。纯粹的时间驱动切片。

每个关闭的切片都会写入一个 Markdown 文件，路径为：

```
memory/episodic/slices/2025/11/21/0825.md
```

路径就是时间戳：年、月、日，以及第一条消息的时-分。文件带有 YAML 前置元数据——关注点、摘要、决策、未决事项、情绪基调，以及标签——全部由代理在每个轮次自动维护。你永远不需要手动标注任何内容。

跨月、跨年地从头读到尾，这个切片目录就是你的自传。可 Git 对比、人类可读、可移植。

## 询问你的过去：观察回忆机制的工作

当你问到涉及过去的事情时——比如"关于那个住房项目我做了什么决定？"或"提醒我上次和 Sarah 聊天是什么时候"——Previously 会运行一个可见的两层回忆机制，在答案上方渲染为可展开的卡片。

1. **Flash 回忆**（约 500 ms，DeepSeek-chat）——扫描最近的切片摘要和线索索引（`memory/episodic/strands.json`）。它返回指向相关切片的**指针**：切片 ID、相关性分数，以及每条命中的原因。这是快速、近似的通道。它同时作为副作用维护当前活动切片的元数据（摘要、标签、未决事项）——无需单独的后处理步骤。

2. **深度回忆**（Pro，主模型）——接收 Flash 的指针，并决定通过 `readMemory` 工具完整读取哪些切片。工具调用显示为标有 `"Recalling in detail..."` 的内联卡片。如果 Flash 没有找到任何内容，模型可以自行浏览目录。

回忆结果在共享的带有历史图标的 `ToolLayout` 卡片内渲染：

- **折叠状态**：显示耗时（例如 "1s"）。
- **展开状态**：显示 Flash 的推理文本、回忆命中列表（切片 ID、原因、相关性百分比），以及它发现相关的标签。

深度回忆期间的工具调用以带有友好标签的内联卡片形式呈现：

| 工具 (Tool) | 你看到的标签 |
|------|---------|
| `readMemory` | Recalling in detail... |
| `listMemory` | Recalling more... |
| `readIndex` | Scanning timeline... |

每张卡片有五种视觉状态：**运行中**（旋转加载器）、**已完成**（绿点）、**出错**（红色）、**中断**（黄色）、**拒绝**（红色）。点击或按 Enter/Space 展开查看完整工具输出——不需要 JS 高度测量，仅使用 CSS `grid-template-rows` 过渡。

消息本身分三个阶段渲染，与代理内部执行顺序一致：**回忆** → **推理** → **响应**。你会看到代理扫描时间轴、思考、然后回答——全部在一条消息中完成。

## 首次运行体验：演示模式

Previously 没有应用内引导向导。取而代之的是，它附带一个 **DEMO_MODE** 环境标志，可将部署转变为只读导览。

在环境中设置 `DEMO_MODE=true`，每次 `memory/` 读取都会被重定向到 `memory/demo/personal_14/`——一个预先植入的虚构角色，跨越 30 多个切片和多年的对话。屏幕顶部的固定横幅告诉你，你所写的内容都不会被保存。写入操作看似成功，但实际上从未写入仓库；刷新页面，一切恢复如初。

在线演示位于 [previously-demo.ldwid.com](https://previously-demo.ldwid.com)。它运行完整的两层回忆系统（Flash + Pro），因此你可以浏览时间轴、翻阅过去的切片、自由聊天，并观察回忆机制的工作——一切都不需要设置你自己的部署。

> **重要提示：演示数据集不在仓库的 `main` 分支上。** 它存在于 `DEMO_REF` 分支，并在 `DEMO_MODE` 开启时通过固定到该分支的 GitHub 读取来提供服务。在本地 `main` 分支上尝试浏览 `memory/demo/personal_14/` 将找不到任何内容。

### 开始你自己的时间轴

要从演示模式切换到自己的时间轴：

1. 使用你自己的 GitHub 仓库和 GitHub 个人访问令牌（contents read/write 权限）部署 Previously。
2. 不设置 `DEMO_MODE`，或将其设置为 `false`。
3. 就是这样。你输入的第一条消息就会打开你的第一个切片。

无需创建账号，无需设置数据库，无需种子数据。时间轴从空白开始。记忆从你第一次对话起逐步积累。

## Strands：跨越时间的语义层

随着切片的积累，代理会构建一个称为 **strands** 的轻量级关键词索引。strand 是一个话题关键词——比如 `work`、`family`、`housing`——它在多个切片中重复出现。文件 `memory/episodic/strands.json` 将每个 strand 映射到包含它的每一个切片。

Flash 在回忆期间扫描 strand 索引，以按主题（而非仅按时间远近）查找切片。如果你在三年间跨越 30 个切片讨论过"housing"，Flash 通过 strand 索引找到它们，然后 Pro 读取实际内容。

```preview
demo: strands-index
```

> **路线图：** 目前 strands 只是一个轻量的关键词到切片的索引。更丰富的、带有滚动摘要和深度回忆集成的一等公民 strand 是未来的里程碑。目前，该索引是无损的——它指向每一个相关切片，Pro 读取它需要的内容。

## 这对日常使用意味着什么

没有任何仪式。你不需要打开"新对话"。你不需要标记或分类任何东西。你不需要管理记忆。

- **出现** —— 打开页面。时间轴就在那里，已经加载了你最近的过去。英雄区显示 "Previously on {你的名字}"。
- **交谈** —— 输入文字并按 Enter。如果已经过了 30 分钟，一个新的切片会自动打开。代理读取时间轴，扫描相关上下文，然后回应。
- **询问过去** —— "关于 X 我做了什么决定？"或"Y 是什么时候发生的？"Flash 扫描摘要和 strand 索引，Pro 读取完整的切片，你会看到工具卡片出现并完成解析。
- **滚动** —— 实时聊天上方的完整时间轴就是你的全部历史。按日期分组，分页加载，带有电影式的"N 分钟/小时/天后"标题卡片，填充最后记录时刻与现在之间的间隔。

这就是整个循环。无需设置，无需清理，无需线程管理。只是一个带记忆的、持续进行的对话。

## 项目状态

**Previously 正处于活跃的早期开发阶段，尚未准备好用于个人或生产环境**（状态：实验性的）。本指南记录了代码中当前存在的 v0.1.0 版本。核心架构是可用的，但许多子系统仍在设计和构建中。本快速入门文档将其定位为个人自托管部署，而非 SaaS 注册服务。
