# 简介

Previously 是一个轻量级云端 Agent，其定义性特征是情景记忆（episodic memory）：它能读、能写、能推理、能行动，并且被设计为能够跨越时间永久记忆（标语："Previously on you."）。

## Previously 是什么

Previously 不是另一个让你在一次性会话中聊天的助手。它是一个围绕单一理念组织的持久化云端 Agent：**一条时间线上的持续关系。** 你是指挥官；Previously 是你的幕僚。你出现，你交谈，你离开。当你回来时——数小时、数天或数周之后——你首先看到的是一条纵向的时间线，而非一列聊天线程。Agent 通过扫描事件发生的*时间*来判断相关性，然后检索*当时*说了什么。

没有对话。没有对话管理。只有一条时间线。

## "Previously on..." 的隐喻

名字源于电视：每集新剧集开始前，一段简短回顾提醒你上次发生了什么——刚好足够你接上剧情。Previously 做的也是同样的事。每次你回来，它都会回顾你自己的时间线，把重要的内容带到前面，让你永远不会从零开始。

> **关键要点：Previously 用时片（time slice）取代了聊天线程模型，使记忆与交互合为一体。修复了记忆模型，就根本无需管理对话。**

## 与聊天线程 Agent 的区别

主流 Agent 将你的生活分割成一个个聊天线程。每个线程从零开始。线程之间的记忆彼此孤立、脆弱且有损——需要外挂向量数据库、RAG 流水线和 prompt 黑客手段来近似实现本应是原生能力的特性。

Previously 用时片**取代**了线程。因为交互历史与记忆共享同一结构——按时间顺序排列的切片序列——跨会话回忆是系统固有的属性，而非事后修补。Agent 不会突然忘记长对话的开头，因为上下文是每个请求动态组装的，而不是累积在逐渐膨胀的 prompt 窗口中。每个请求的负载保持有界且相关。

## 词汇巡礼

三个术语承载了整个架构。

**Slice（切片）** —— 一次对话片段。时间线上位于 `memory/episodic/slices/YYYY/MM/DD/HHMM.md` 的一个 Markdown 文件。你开始交谈时打开，保持活跃期间持续存在，30 分钟无操作后自动关闭。携带结构化的 YAML frontmatter：焦点、摘要、决策、未决事项、标签、情绪基调。Slice = *发生了什么，以及何时发生。*

```preview
demo: slice-file
```

**Strand（线索）** —— 跨多个切片重复出现的关键词（如 `health`、`work`、`housing`）。索引文件 `memory/episodic/strands.json` 将每个关键词映射到所有携带该标签的切片路径。Strands 在切片关闭时基于其标签构建。Slice = 发生了什么；strand = 它关于什么。Strands 构成了时间线上轻量、无损的语义层。

**Recall（回忆）** —— 双层检索引擎。**Flash**（DeepSeek-chat）在一次快速往返中扫描近期已关闭切片的摘要，返回最多五个带相关度分数的指针，并在同一次调用中完成意图分类和元数据维护。**Pro**（主模型，启用思考模式）接收这些指针，然后通过 `readMemory` 工具完整读取切片——这是代价较高、更彻底的扫描。Flash 被预期可能存在不完美；当 Flash 没有返回结果时，Pro 会做更深层的工作。

| 术语 | 层级 | 角色 | 速度 |
|------|------|------|------|
| Slice | 情景层（何时） | 记录发生了什么 | 存储，非检索 |
| Strand | 语义层（什么） | 跨切片的关键词 | 切片关闭时构建 |
| Flash recall | 检索层 1 | 扫描 + 指针 + 维护 | 约 500 ms（典型） |
| Pro recall | 检索层 2 | 深度读取 + 推理 | 数秒（每次工具调用） |

## 研究基础

情景/语义的二分并非随意之举。它基于 Endel Tulving 的情景记忆与语义记忆理论（1972）以及情境依赖记忆研究（Godden & Baddeley 1975；Smith & Vela 2001）。Slices 编码了*何时*（情景层）。Strands 和记忆节点编码了*什么*（语义层）。Recall 遵循同样的顺序：先扫描何时，再检索什么。

## 状态

Previously 目前是 **EXPERIMENTAL（实验阶段）**——早期开发，尚未准备好用于个人或生产环境。这是一个单人研究项目，但计划长期维护。目前已可用的功能包括：流式聊天与工具调用可见性、时片存储与自动 30 分钟静默切片、Flash recall 扫描与深度 recall、每轮元数据维护、带历史淡出效果的时间线 UI、记忆节点、带路径白名单的 GitHub 文件工具、多模型支持，以及中英双语国际化。

Strand 索引在切片关闭时构建，Pro 可以按需读取，但 Flash 扫描尚未自动使用 strand 索引——它只扫描近期切片的摘要。一个拥有自身滚动摘要并与 recall 深度集成的"一等公民" strand 是未来的里程碑。
